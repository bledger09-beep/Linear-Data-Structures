# Linear-Data-Structures
//1. A developer would use a linked list when they need efficient insertion and deletion of elements, especially when the size of the data can change frequently. Unlike arrays, linked lists do not require shifting elements when adding or removing items, which can improve performance in these situations. They are also useful when memory usage needs to be dynamic, as linked lists allocate memory as needed rather than requiring a fixed size upfront. This makes them a good choice for applications like managing queues, stacks, or dynamic data collections.

//2. A stack is a data structure that follows the Last-In, First-Out (LIFO) principle, meaning the most recently added item is the first to be removed. A queue follows the First-In, First-Out (FIFO) principle, meaning the first item added is the first one to be removed.

//3. A developer would use a stack when they need to keep track of operations in reverse order, such as implementing an "undo" feature in a text editor. Each action the user performs is pushed onto the stack, and when the user clicks undo, the most recent action is popped off and reversed. Stacks are also useful for managing function calls in programming, since the most recent function call must finish before earlier ones can resume.

//4. A developer would use an array instead of a list when the size of the data is known in advance and will not change during the program. Arrays are more memory-efficient and can offer slightly 
faster access times because their size is fixed and stored in a continuous block of memory.
